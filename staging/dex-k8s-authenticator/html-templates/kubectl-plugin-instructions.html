Configure kubect to Use the Plugin

These commands will crate a new kubeconfig context that instruct kubectl to delegate authentication tasks to the Konvoy credentials plugin

Create a cluster configuration


{{- if .caPem }}
  kubectl config set-cluster {{ .clusterName }} \
      --certificate-authority=${HOME}/.kube/certs/{{ .clusterName }}/k8s-ca.crt \
      --server={{ .kubeAPI }}
{{- else }}
  kubectl config set-cluster {{ .clusterName }} \
      --server={{ .kubeAPI }}
{{- end }}

Create a kubeconfig User Profile

If you plan on authenticating to the same cluster with multiple users, you should created multiple user profiles

kubectl config set-credentials {{ .profileName }} \
      --exec-command={{ .runPath }} \
      --exec-api-version=client.authentication.k8s.io/v1beta1 \
      --exec-arg="-auth-url={{ .asyncAuthURL }}" \
      --exec-arg="-kubeconfig-user={{ .profileName }}" \
      --exec-arg="-ca-file=${HOME}/.kube/certs/{{ .clusterName }}/k8s-ca.crt"

Create the context

A kubeconfig context is a cluster and user profile

kubectl config set-context {{ .clusterName }} \
      --cluster={{ .clusterName }} \
      --user={{ .profileName }}

Set the active context

kubectl config use-context {{ .clusterName }}
